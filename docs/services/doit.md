# DoIt

**URL:** `http://tasks.homelab`
**Profile:** core (always running)
**Image:** `ghcr.io/standw7/doit:latest` (frontend), `ghcr.io/standw7/doit-backend:latest` (backend)

## What It Does

A daily task management app with calendar integration. Built with Next.js (frontend) and FastAPI (backend) with SQLite storage.

## Required Environment Variables

| Variable | Description | How to Get |
|----------|-------------|-----------|
| `DOIT_SECRET_KEY` | JWT signing secret | Auto-generated by setup script, or `openssl rand -hex 32` |
| `DOIT_INTERNAL_API_KEY` | API key for Telegram bot integration | Auto-generated by setup script, or `openssl rand -hex 32` |
| `DOIT_GOOGLE_CLIENT_ID` | Google OAuth Client ID (optional, for calendar) | [Google Cloud Console](https://console.cloud.google.com) |
| `DOIT_GOOGLE_CLIENT_SECRET` | Google OAuth Client Secret (optional) | Same as above |

## First-Time Setup

1. Start the stack: `docker compose up -d`
2. Open `http://tasks.homelab`
3. Sign up with an email and password
4. (Optional) Go to Settings â†’ Connect Google Calendar for calendar sync

### Google Calendar Integration (Optional)

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project or select an existing one
3. Enable the Google Calendar API
4. Create OAuth 2.0 credentials:
   - Application type: Web application
   - Authorized redirect URI: `http://tasks.homelab/settings?google=callback`
5. Copy Client ID and Client Secret to your `.env` file
6. Restart: `docker compose restart doit-backend doit`

## Configuration

**Data:** Stored in Docker volume `doit_db` (SQLite database)

## Troubleshooting

**Can't sign up:** Check backend logs:
```bash
docker compose logs doit-backend
```

**Google Calendar not syncing:** Verify the redirect URI matches exactly. Check that the Google Calendar API is enabled in your Google Cloud project.
